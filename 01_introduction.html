The role of computing in scienceÂ¶

  • Science has traditionally been divided into experimental and theoretical disciplines
  • Computing is emerging as an important part of science -> a "third branch"
    • related to theory ("prove it" computationally)
    • related to experiments ("simulate" a world with a known answer)

Openness in ScienceÂ¶

  • Established code of conduct
    • Theoretical work: publish derivations and proofs
    • Experimental work: publish methods used and results (with detailed experimental data available upon request)
    • University policies [1] [2]
  • Computational work does not have established rules
    • Rare to publish source code -> considered a competitive advantage (or unnecessary)
    • Starting to see editorials [1] [2] and conference sessions [1] [2] calling for increased openness
  • How to do it?
    • During peer review
    • In perpetuity (software dependencies, hardware needs, etc.)

Requirements on Scientific ComputingÂ¶

Computational ThinkingÂ¶

"Informally we might define CT as a set of thinking or problem-solving strategies that are used to solve problems when working with computers, and that can also be applied to many problem types, even absent the computer."

"Formulating problems in a way that enables us to use a computer and other tools to help solve them."

6 Algorithms That Can Improve Your LifeÂ¶

Listen to the story in full. Think hard about the computer science part of each of these algorithms. http://www.wnyc.org/story/algorithms-practical-efficiency

  • Decomposition: The ability to break down a task into minute details so that we can clearly explain a process to another person or to a computer, or even to just write notes for ourselves. Decomposing a problem frequently leads to pattern recognition and generalization, and thus the ability to design an algorithm.
  • Pattern Recognition: The ability to notice similarities or common differences that will help us make predictions or lead us to shortcuts. Pattern recognition is frequently the basis for solving problems and designing algorithms.
  • Pattern Generalization and Abstraction: The ability to filter out information that is not necessary to solve a certain type of problem and generalize the information that is necessary. Pattern generalization and abstraction allows us to represent an idea or a process in general terms (e.g., variables) so that we can use it to solve other problems that are similar in nature.
  • Algorithm Design: The ability to develop a step-by-step strategy for solving a problem. Algorithm design is often based on the decomposition of a problem and the identification of patterns that help to solve the problem. In computer science as well as in mathematics, algorithms are often written abstractly, utilizing variables in place of specific numbers.

What is Python?Â¶

  • Simple, clean syntax
  • Easy to learn (relative to other programming languages)[1]
  • Object oriented
  • Interpreted: no need to compile code
  • Dynamically typed: no need to define the type of variables
  • Automatic memory management
  • Interactive
  • Runs on all major operating systems (Windows, Mac, Linux)
  • Open source (free)
  • Expressive: do more with fewer lines of code
  • Lean: modules

What can you do with Python?Â¶

  • OS support: manage files and directories
  • Glue existing applications together
  • Scientific abstraction
    • access to powerful C and Fortran libraries
    • parallel
  • Data Analysis
  • Visualization
  • Build GUIs
  • Scrape websites
  • Build websites
  • Anything!

PerformanceÂ¶

The greatest performance improvement comes from going to the working state from the non-working state

"CPU time is cheap, my time is expensive"

  • Expressive: minimal time required to develop code
  • Compact (20,000 lines in Python is approximately 100,000 lines in Java or 200,000 lines C)
  • Interpreted and dynamically typed means it is slower
    • Is this a problem if the code takes a few seconds to run?
    • Use optimized libraries: e.g. numpy, scipy
    • Find the bottleneck and write it in C/C++/Fortran: e.g. cython
  • Implementing the built-in Python modules would require some advanced programming skills in other languages

ModulesÂ¶

  • A large standard (built-in) library
  • A large collection of add-on packages
    • Numerical computing: Numpy, Scipy (BLAS and LAPACK)
    • Visualization: Matplotlib, Seaborn (links to R)
    • Data wrangling: Pandas
    • Parallel Computing: IPython, multiprocessing, mpi4Py
    • Performance: cProfile
    • Mixed Language Programming: C, C++, Fortran
    • Database: SQLite, SQLAlchemy
    • Web: Django
    • GIS and spatial analysis: PySAL, shapely, geopandas, cartopy
    • Optimization: cplex, gurobi
